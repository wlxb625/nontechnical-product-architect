# 阶段质量门槛

## Gate 1：产品定义

必须回答：

- 谁是核心用户？
- 在什么场景使用？
- 解决什么明确问题？
- 用户当前如何解决？
- 产品的核心价值是什么？

未通过表现：只有功能列表，或“给所有人使用”。

## Gate 2：MVP

必须满足：

- 只有一个主要价值闭环；
- 每个必需功能都有明确作用；
- 已记录暂缓功能和原因；
- 有可观察的成功标准；
- 第一版规模与个人或小团队资源匹配。

## Gate 3：用户流程与页面

必须满足：

- 用户知道从哪里开始；
- 核心操作可以完成；
- 成功、空、加载和失败状态存在；
- 数据保存结果明确；
- 用户知道下一步；
- 页面没有明显孤岛和重复。

## Gate 4：数据与 AI

必须回答：

- 保存哪些数据？
- 谁能读取和修改？
- 用户如何删除？
- AI 每次读取什么？
- AI 输出是否直接产生重要影响？
- AI 失败时如何降级？
- 是否区分原始记录、摘要和长期记忆？

## Gate 5：技术与风险

必须满足：

- 技术由需求推导；
- 没有未经证明的复杂基础设施；
- 主要费用来源已说明；
- 关键第三方依赖已说明；
- 地区、隐私和安全风险已记录；
- 用户能用普通语言解释主要技术选择。

### Gate 5A：现实可用性硬门槛

每个关键第三方服务必须回答：

- 它是什么，在当前产品中负责什么？
- 用户所在地区能否注册和稳定访问？
- 产品目标用户能否稳定访问？
- 免费版只适合测试，还是能支撑正式上线？
- 付费支持什么卡或支付渠道，使用什么币种？
- 用户当前是否具备这种付款方式？
- 从注册到部署大致需要哪些步骤？
- 是否需要实名认证、企业主体、域名备案或应用审核？
- 超额、欠费或免费额度结束后会发生什么？
- 是否提供至少一个符合用户地区和付款条件的替代方案？

出现以下任一情况时，不得把方案标记为“推荐使用”：

- 用户明确没有该服务支持的付款方式；
- 目标用户所在地区访问明显不稳定；
- 正式上线必须付费，但付款能力未确认；
- 需要实名认证、备案或企业主体，但用户无法满足且没有替代路线；
- 仅凭免费额度作出长期生产方案；
- 无法解释该服务的用途和基本使用流程。

当用户位于中国大陆且没有境外支付卡时，只支持境外信用卡持续付费的服务最多标为“仅适合免费原型”，不能作为默认生产环境核心依赖。

## Gate 6：Codex 交接

必须满足：

- 每个任务可独立完成和验收；
- 任务有前置条件；
- 任务说明不应修改的范围；
- 任务包含失败处理；
- 关键步骤设有暂停检查点；
- 不要求 Agent 一次完成整个产品；
- 第一批任务能得到可运行的最小结果；
- 部署、数据库、AI API 等任务使用的服务已经通过 Gate 5A；
- 环境变量清单不包含用户无法申请或无法付费的服务。
